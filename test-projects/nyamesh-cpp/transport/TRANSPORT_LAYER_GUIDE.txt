===========================================
📁 Transport Files and Their Roles
===========================================

1. 📦 InProcessTransport.cpp/h (262行)
   ━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━
   Role: プロセス内メッセージバス（基盤）
   Purpose: 同一プロセス内でのノード間通信
   
   🎯 When to use:
   - テスト環境
   - 単一プロセス内の分散アーキテクチャ
   - 開発・デバッグ時
   
   🔧 Key Features:
   - InProcessMessageBus singleton
   - ノード間ルーティング・ブロードキャスト
   - メッセージキューイング
   - 統計情報収集
   
   💡 Example Usage:
   auto transport = createInProcessTransport("node1");
   transport->initialize();
   transport->send(message);

2. 🌐 NyaMesh2TransportAdapter.cpp/h (465行)
   ━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━
   Role: 軽量P2Pアダプター（単一トランスポート）
   Purpose: InProcessTransportを使った基本P2P機能
   
   🎯 When to use:
   - シンプルなP2P通信
   - 軽量なアプリケーション
   - InProcessTransportの拡張が必要な場合
   
   🔧 Key Features:
   - SafeMessage形式の変換
   - 用途別最適化設定
   - 非同期操作サポート
   - JavaScript版互換API
   
   💡 Example Usage:
   auto adapter = createNyaMesh2TransportAdapter("node1", true);
   adapter->configure(createRequirementsForTextEditing());
   adapter->sendToPeer(peerId, message);

3. 🚀 P2PTransportAdapter.cpp/h (701行)
   ━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━
   Role: ハイブリッドP2Pシステム（革命的）
   Purpose: NyaMesh2とMillionPeerを自動選択する最高性能システム
   
   🎯 When to use:
   - 本格的なP2P通信が必要
   - 大規模・高性能が要求される
   - 用途に応じた自動最適化が欲しい
   
   🔧 Key Features:
   - NyaMesh2 + MillionPeer 自動切り替え
   - 用途別最適化（text-editing, file-sharing, search）
   - パフォーマンス監視・自動切り替え
   - リアルタイム統計情報
   
   💡 Example Usage:
   auto adapter = createP2PTransportAdapter();
   adapter->configure(createRequirementsForTextEditing());
   adapter->broadcastMessage(message);

===========================================
🎯 Transport Selection Guide
===========================================

選択基準:

🔹 Development/Testing:
   → InProcessTransport

🔹 Simple P2P Applications:
   → NyaMesh2TransportAdapter

🔹 High-Performance P2P Applications:
   → P2PTransportAdapter (推奨)

🔹 Text Editor (リアルタイム協調):
   → P2PTransportAdapter with text-editing requirements

🔹 File Sharing (大規模):
   → P2PTransportAdapter with file-sharing requirements

🔹 Search (分散検索):
   → P2PTransportAdapter with search requirements

===========================================
🧪 Performance Characteristics
===========================================

InProcessTransport:
- Latency: <1ms (same process)
- Throughput: 100k+ messages/sec
- Memory: Minimal
- Scale: Single process only

NyaMesh2TransportAdapter:
- Latency: 5-10ms
- Throughput: 10k+ messages/sec
- Memory: Low
- Scale: Small networks (< 100 nodes)

P2PTransportAdapter:
- Latency: 5ms (NyaMesh2) / 50ms (MillionPeer)
- Throughput: 50k+ messages/sec
- Memory: Medium
- Scale: Up to 1M nodes (MillionPeer)

===========================================
🚨 Important Notes for Claude Code
===========================================

1. 🔄 ALWAYS use P2PTransportAdapter for real applications
   - It automatically selects the best transport
   - Provides the highest performance
   - Has the most features

2. 📝 JavaScript Compatibility:
   - All transports support JavaScript-compatible APIs
   - mesh.send() and mesh.publish() patterns work
   - SafeMessage format is standardized

3. 🛠️ Build Dependencies:
   - All transports are header-included in nyamesh2
   - No external dependencies required
   - Thread-safe implementations

4. 🎮 Testing Strategy:
   - Use InProcessTransport for unit tests
   - Use NyaMesh2TransportAdapter for integration tests
   - Use P2PTransportAdapter for production tests

===========================================
📋 Quick Reference Commands
===========================================

Build with Transport Layer:
```bash
# Include all transport sources in CMakeLists.txt:
src/nyamesh2/transport/InProcessTransport.cpp
src/nyamesh2/transport/NyaMesh2TransportAdapter.cpp
src/nyamesh2/transport/P2PTransportAdapter.cpp
```

Example CMakeLists.txt entry:
```cmake
set(TRANSPORT_SOURCES
    src/nyamesh2/transport/InProcessTransport.cpp
    src/nyamesh2/transport/NyaMesh2TransportAdapter.cpp
    src/nyamesh2/transport/P2PTransportAdapter.cpp
)
```
